React-Router v1.0.0-rc1 Async Routes

One of the game changers in the new react-router release, is being able to define asynchronous route definitions for splitting up your code, enabling lazy loading of your components.

Routes may define getChildRoutes and getComponents methods, to specify a method of retrieving your modules, as we'll see.

Their example uses a declarative Object literal syntax, which I think can be improved upon:

var CourseRoute = {  
  path: 'course/:courseId',

  getChildRoutes(location, callback) {
    require.ensure([], function (require) {
      callback(null, [
        require('./routes/Announcements'),
        require('./routes/Assignments'),
        require('./routes/Grades'),
      ])
    })
  },

  getComponents(callback) {
    require.ensure([], function (require) {
      callback(null, require('./components/Course'))
    })
  }
};

While this ticks most of the boxes, it means changing your usual usage somewhat. Abstracting this away in say, into a dedicated class not only makes it more re-usable, but I think cleaner when implementing with React's render syntax:

import {Route} from 'react-router';

export default class AsyncRoute extends Route {}

AsyncRoute.defaultProps = {  
    getComponents(location, callback) {
        System.import(this.async).then((mod) => {
            callback(null, mod.default);
        });
    },
    getChildRoutes (location, callback) {
        callback(null, this.children || []);
    }
}

As you can see, extending the react-router's Route class to encapsulate the logic, has a lot of benefits.

We've internalized the logic making it far more portable, as well as defining the contract for module loading in one place.

Here I'm using the SystemJS module loader, to dynamically load the Route's component when required.

Their example uses Webpack's require.ensure mechanism, although you could swap this out for any module loader prefered.

class SomeView extends React.Component {  
   render() {
      return (
         <Router history={history}>
            <Route path='/' component={HomeView}/>
            <AsyncRoute path='login' async="some/path/LoginView.js"/>
         </Router>
      )
   }
}
