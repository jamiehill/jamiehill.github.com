GlobalRemoteObjectFaultHandler woes

Honestly, the trouble I’ve had trying to get Parsley’s GlobalRemoteObjectFaultHandler working, you wouldn’t believe. The documentation on the Spicefactory site is blessedly comprehensive (well at least to a certain level), but it can be frustrating trying to find specific use-case examples when the documentation falls short. (Sorry Jens, criticism is not my intention!).

Fortunately, this post saved my bacon.

I had done just that – adding the CairngormIntegrationSupport tag to the ContextBuilder, but my downfail was the cairngorm namespace declaration being used. I’d previously had Cairngorm Popup Library functionality imported, that had automatically added the following namespace for Cairngorm:

http://ns.adobe.com/cairngorm

But as the post above describes, to get the Cairngorm integration working, the Cairngorm namespace required is:

com.adobe.cairngorm.*

Thanks Adobe for suffering me hours of head scratching!
