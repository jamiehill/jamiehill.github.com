Flex Spark Skin Transition to Animate Boolean Properties

There may be a better/more ‘Spark’ way of doing this but at the moment I can’t find how. When entering a certain state in my SparkSkin, I needed to flash a couple of LEDs. The LED’d are basically ToggleButtons, so we basically just need to ‘toggle’ the ‘selected’ property at regular intervals.

The final solution proved a little more fiddly, but the result works well.

[SWF]/wp-content/uploads/2011/12/FlightDeckPanel.swf, 477, 175[/SWF]

The Solution

Using the SetAction Effect initially seemed like the way to go, but it hit me that this only seemed appropriate when combining in Sequence with an actual ‘visiual’ IEffect instance.

The SetAction Effect, does what it says on the tin, it’s simpl an action that sets a property. It doesn’t have a duration, or repeatCount etc. that other IEffects have. As such, the only way of achieving a sequence of SetActions, at specified intervals, would be to combine it with a ‘dummy’ Effect…

```

But using ‘dummy’ effects, just to achieve our goal, is a hackish way of doing things. After considering different options, it occurred to me that similar functionality could be achieved using the Animate IEffect.

The Animate IEffect offers all of the functionality to specify a duration and repeat behaviour that I required, with the properties needing to be ‘Animated’ specified using a SimpleMotionPath instance. Bingo!

But no, this didn’t seem to work either, and upon further investigation, it seemed that Boolean properties just weren’t rendered correctly. The default interpolation method used by the Animate class is the NumberInterpolator – which provides interpolation between start and end values represented as Number instances. Now Boolean true and false can be represented as Number values 1 and 0, so I would have expected it to work. But the interpolated number sequence when interpolating between 1 and 0 would obviously pose a problem, when my component was expecting a simple 1 or 0. Which led me to the idea of a BooleanInterpolator.

The BooleanInterpolator  
-----------------------

I won’t expand on the implementation here, as it’s not necessary. I simply implemented the IInterpolator interface, and fixed the interpolated values to either 1 or 0, and it worked!

You can [download the BooleanInterpolator here](http://blog.tiltdigital.com/wp-content/uploads/2011/12/BooleanInterpolator.as).

So, there we have it. A way of Animating between Boolean values, suitable for use in Spark Transitions.
