Cufon Font Replacement

For nice font replacement on this site, I’m using the Cufon fast text rendering engine. After a fair bit of research, having previously used both sIFR and typeface.js, it does feel particularly easy to use, and as per the authors claims:

No plug-ins are required - it only uses features natively supported by the client

Compatibility - it works on every major browser on the market

Ease of use - no or near-zero configuration needed for standard use cases

Speed - it’s very fast, even for sufficiently large amounts of text

At first, the setup was incredibly easy to manage. After only a few minutes, I generated the font I wanted to use, included the necessary javascript files in my page, and had replaced certain css selectors’ text with the generated font – Very happy indeed!

Then my first problem hit me. Getting element ‘:hover’s to work just wouldn’t work for me. After much Googling, I narrowed the required code down to the following:

Cufon.replace('h2 a', {  
  hover: 'true'
});

But would this do what I wanted? Of course it wouldn’t. Cufon either replaced the required selector with the generated font, but the hover color change would fail, or vice versa, The hover worked, but the font wasn’t replaced.

Hmmm :/

After much much more Googling, the answer was to check the DOCTYPE I was using: a transitional one. I found switching to the Strict version did the trick, ie:
