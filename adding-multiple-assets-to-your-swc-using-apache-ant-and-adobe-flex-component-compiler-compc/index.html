Adding Multiple Assets To Your Swc Using Apache ANT And Adobe Flex Component Compiler (Compc)

This is a topic I felt I should cover, as after many hours spent Googling, I still haven’t found a viable solution, and neither does it seem, have scores of other Flex Developers.

If you need to package an arbitary set of asset files in a swc, the process is not only laborious, but actually unworkable. The Compc Ant task, provides the ‘include-file’ property, that lets you explicitly tell the compiler, that that file should be included in the resultant swc. On large scale enterprise applications – that may contain hundreds of asset files – there is no way of dynamically including the contents of a certain folder.

<include-file name="an_image.png" value="path/to/image/folder/an_image.png"/>

After scratching my head for a considerable time, extending Compc and adding an easier mechanism to do this, seemed like the only option. So here it is – Compcx (CompcExtended).
