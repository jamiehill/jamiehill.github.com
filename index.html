Enabling Babel ES7 Features in SystemJS/JSPM

Enabling ES7 support in SystemJS, is a pretty trivial thing to do, but took me a while to figure out how.

I'll cut to the chase and show you how:

  "transpiler": "babel",
  "babelOptions": {

        stage: 0,

    ]
  },

React-Router v1.0.0-rc1 Async Routes

One of the game changers in the new react-router release, is being able to define asynchronous route definitions for splitting up your code, enabling lazy loading of your components.

Routes may define getChildRoutes and getComponents methods, to specify a method of retrieving your modules, as we'll see.

Their example uses a declarative Object literal syntax, which I think can be improved upon:

var CourseRoute = {  
  path: 'course/:courseId',

  getChildRoutes(location, callback) {
    require.ensure([], function (require) {
      callback(null, [
        require('./routes/Announcements'),
        require('./routes/Assignments'),
        require('./routes/Grades'),
      ])
    })
  },

  getComponents(callback) {
    require.ensure([], function (require) {
      callback(null, require('./components/Course'))
    })
  }
};

While this ticks most of the boxes, it means changing your usual usage somewhat. Abstracting this away in say, into a dedicated class not only makes it more re-usable, but I think cleaner when implementing with React's render syntax:

import {Route} from 'react-router';

export default class AsyncRoute extends Route {}

AsyncRoute.defaultProps = {  
    getComponents(location, callback) {
        System.import(this.async).then((mod) => {
            callback(null, mod.default);
        });
    },
    getChildRoutes (location, callback) {
        callback(null, this.children || []);
    }
}

As you can see, extending the react-router's Route class to encapsulate the logic, has a lot of benefits.

Routing Overloaded Controller Actions in Play Framework 2

Came across an interesting dilema whilst trying to wire up a tree control using Play Framework 2.

The controls api only provides one method for retrieval of initial remote data and also subsequent loading of on-demand data for updating the control. So, how to wrap two remote requests using the same controller action? Overload the controller method! Er … not quite that simple …!

In Play, you can’t – for obvious reasons – map the same url to different actions. This still applies to overloaded methods, as the framework still sees them as two separate methods, not simply two versions of the same thing!

# Routes
GET      /getData      controllers.Application.getData()  
GET      /getData      controllers.Application.getData(nodeId)  

#Controller
public static Result getData() {  
    // load initial remote data to populate the control
}

public static Result getData(String nodeId) {  
    // load on-demand data to update control
}

#Controller
public static Result getData(String id) {  
    if (id == "root")
        return loadRootNodes();
    return loadNode(id);
}

    // load initial remote data to populate the control

Adding Multiple Assets To Your Swc Using Apache ANT And Adobe Flex Component Compiler (Compc)

This is a topic I felt I should cover, as after many hours spent Googling, I still haven’t found a viable solution, and neither does it seem, have scores of other Flex Developers.

If you need to package an arbitary set of asset files in a swc, the process is not only laborious, but actually unworkable. The Compc Ant task, provides the ‘include-file’ property, that lets you explicitly tell the compiler, that that file should be included in the resultant swc. On large scale enterprise applications – that may contain hundreds of asset files – there is no way of dynamically including the contents of a certain folder.

<include-file name="an_image.png" value="path/to/image/folder/an_image.png"/>
